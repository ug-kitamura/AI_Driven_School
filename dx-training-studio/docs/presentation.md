プレゼン原稿
    1. はじめに
        私が作ったのは、DX ツールのトレーニングコンテンツを作成する、４ペイン ワークスペースです。
        ツール名はそのままで 「DX Training Studio」 です。
    2. 解くべき課題
        私がこのスクールに入ったのも、この課題を解決したかったからです。
        会社で新しいプロジェクトが始まり、ツール環境が git / GitHub に変わりました。
        でも、ほとんどのメンバーは git / GitHub に慣れていなくて、理解度もバラバラです。
        一方で、教えられる人のリソースは限られていて、全員に手取り足取り教えるのは無理です。
        なので 「受講者が自分のペースで独習できるトレーニングコンテンツを作ること」 を目標としました。
        ただ、コンテンツを作ること自体面倒なので、このツールを作ることにしました。
    3. ツール説明
        実際の画面をお見せします。
        「本気 AI ドリル」を参考にして、トレーニングをシリーズ → コース → レッスン の 3 階層に分解しています。
        それぞれを 4 つのペインで管理します。
        ペイン 1
            シリーズとコースの管理です。
            追加・削除・並べ替えができて、進捗も表示します。
        ペイン 2 
            レッスンの管理ですが、ここでは受講の順番を細かく設定できます。
            複数のコースを修了したら次に進める「合流」、
            修了後に複数のコースに分岐させる「分岐」、
            シリーズをまたいだ複雑なフローを設定できます。
            これを 「DX トレーニング曼陀羅」 と呼んでいます。
        ペイン 3 
            原稿のエディタです。
            マークダウンと YAML で書けて、プレビューと差分表示も切り替えられます。
        ペイン 4 
            画像マネージャーです。
            AI に作らせたり、ウェブ検索でフリー画像を取ってきたりできます。
    4. 工夫した点
        一番力を入れたのが AI による画像プロンプトの自動生成 です。
        プロンプトを記入すれば AI が画像を作ったり検索したりするのですが、
        さらにそこから一歩進んで、原稿内容とカーソル位置から、AI がプロンプト自体を自分で作ります。
        ユーザーはボタンを押すだけで、文脈に合った画像を用意できます。
        これはかなり気に入っています。
    5. 苦労した点
        一番苦労したのは DX トレーニング曼陀羅の実装です。
        コースの並べ替えと曼陀羅の図が連動しないといけない
        入口を変えたら出口も変わる
        矢印が二重に描画されるのを防ぐ
        循環フローになるのを防ぐ
        などなど...
        考えることがたくさんあって、これが一番しんどかったです。
    6. さいごに
        TypeScript は完全に知識ゼロだったのですが、AI の指示だけで作り上げることができました。
        正直すごく大変でしたが、、大きな自信になりました。
        今後は、トレーニングの原稿の大枠も AI に作らせるようにしたいです。
        おそらく 5 ヶ月目の課題でやることになると思うので、引き続き頑張りたいと思います。
        ありがとうございました！
