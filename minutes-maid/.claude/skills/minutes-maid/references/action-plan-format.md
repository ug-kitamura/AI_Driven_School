# アクションプラン抽出ガイド

## 抽出の基本ルール

アクションプランは「**誰が・いつまでに・何をするか**」の3点が揃って初めて完成。1点でも欠けたら「要確認」フラグを立てる。

---

## 文字起こしからの抽出パターン

### 明示的なアクション（そのまま抽出）

```
「〇〇さん、来月末までに△△をお願いします」
「□□については××さんが担当して、次回会議で報告」
「今月中に全員でアンケートに回答」
```

### 暗黙のアクション（推測して抽出、要確認フラグ）

```
「誰かが対応しないといけませんね」        → 担当：要確認
「なるべく早めにやりましょう」            → 期限：要確認
「このままにしておくわけにはいかない」    → アクション化が必要
```

---

## 要確認フラグの判定基準

| 状況 | フラグ | 対応 |
|------|-------|------|
| 担当者が明示されている | なし | そのまま記録 |
| 担当者が「〇〇チーム」など曖昧 | △ 要確認 | チーム名を記録し、個人名は要確認 |
| 担当者が不明 | ● 要確認 | 赤/オレンジでハイライト |
| 期限が明示されている | なし | そのまま記録 |
| 期限が「なるべく早く」「来月中」など曖昧 | △ 要確認 | 曖昧な言葉を記録し、要確認 |
| 期限が不明 | ● 要確認 | 赤/オレンジでハイライト |

---

## アクション分類（参考情報）

> **注意**: 以下のカテゴリは HTML 出力には反映しない。議事録の性質を理解するための参考情報として活用すること。

| カテゴリ | 例 |
|--------|-----|
| 報告・確認 | 次回会議での報告、関係者への連絡 |
| 資料作成 | 提案書、マニュアル、データ集計 |
| 実施・対応 | ツール導入、プロセス変更、設備対応 |
| 検討・調査 | 課題の調査、選択肢の比較検討 |

---

## 重複・類似アクションのチェック

同一担当者・同一期限のアクションが複数ある場合、まとめるか分けるかを判断：

- **まとめる**: 同じタスクの一部なら1件にまとめる
- **分ける**: 性質が違うなら別々に記録（担当者が混同しないように）

---

## アクションプラン表示例

### 担当者・期限あり（標準）

```
タスク：〇〇の実施計画書を作成する
担当：田中さん  |  期限：4月15日
```

### 担当者あり・期限なし（要確認）

```
タスク：〇〇ツールの動作確認
担当：鈴木さん  |  期限：[要確認] 会議では未設定
```

### 担当者なし（要確認）

```
タスク：〇〇に関する調査
担当：[要確認] 会議では未決定  |  期限：5月末
```

---

## 注意：音声文字起こし特有の問題

| 問題 | 対処 |
|-----|------|
| 同姓の人が複数いる | 発言者情報・文脈から特定、不明なら要確認 |
| 名前が聞き取りにくく誤変換 | 前後の文脈で判断、不明なら要確認 |
| 複数人が同時に発言 | どちらが担当か不明なら要確認 |
| 口語・省略で内容が不明確 | 文脈から補完、補完した部分は「※推測」と記載 |
