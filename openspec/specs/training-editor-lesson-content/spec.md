## Purpose

## Requirements
### Requirement: レッスンコンテンツはフロントマターと本文に分離する

各レッスンの `content` は、YAML フロントマター（`---` で囲む）と Markdown 本文から構成されなければならない（SHALL）。ワークスペース UI の編集モードでは本文のみを表示・編集し、フロントマターを textarea に表示してはならない（MUST NOT）。保存時およびメタ編集保存時、エディタは正規化されたメタと現在の本文から `content` を再生成しなければならない（SHALL）。

#### Scenario: 編集モードにフロントマターが表示されない

- **WHEN** ユーザーが Pane3 の編集モードを開く
- **THEN** エディタ領域には本文のみが表示される
- **AND** `---` で始まる YAML ブロックは表示されない

#### Scenario: 本文編集後に content が再構成される

- **WHEN** ユーザーが編集モードで本文を変更する
- **THEN** 永続化用の `content` には正規化されたフロントマターと更新後の本文が含まれる

### Requirement: フロントマターの正規化と親による上書き

読み込み時およびメタ保存時、エディタはフロントマターを正規化しなければならない（SHALL）。`series` および `course` は、当該レッスンの親シリーズ名・親コース名（ワークスペースのツリー）で上書きしなければならない（SHALL）。フロントマターの値が親と異なる場合も、保存時は親の日本語名が優先されなければならない（MUST）。

#### Scenario: 親のシリーズ名でフロントマターが上書きされる

- **WHEN** フロントマターの `series` が親シリーズ名と異なるレッスンを読み込む
- **THEN** 正規化後のメタの `series` は親シリーズの日本語名と一致する

#### Scenario: 欠損フィールドに初期値が入る

- **WHEN** フロントマターに `tags` が無い、または `estimated_minutes` が不正である
- **THEN** `tags` は空配列、`estimated_minutes` は 0 に正規化される（範囲外の数値は 0–180 にクランプ）

#### Scenario: レガシーステータス draft の移行

- **WHEN** フロントマターまたは `Lesson.status` が `draft` である
- **THEN** 正規化後のステータスは `open` となる

### Requirement: レッスンメタ編集モーダル

Pane3 ヘッダー最右の操作から、レッスンメタ編集ダイアログを開けなければならない（SHALL）。ダイアログは次の項目を提供しなければならない（SHALL）: シリーズ（読取・1件・日本語名）; コース（読取・1件・日本語名）; レッスン（編集可・単一行）; ステータス（`open` / `in_progress` / `done` から選択・アイコン表示・既定 `open`）; 説明（編集可・単一行）; タグ（編集可・複数・各要素は小文字英字・数字・ハイフンのみ）; 推定所要時間（分）（0–180 の整数・既定 0）; 著者（編集可・単一行・既定空）。

#### Scenario: シリーズとコースは編集できない

- **WHEN** レッスンメタ編集ダイアログを開く
- **THEN** シリーズとコースは読取表示のみである
- **AND** 表示名はワークスペース上の親の日本語名である

#### Scenario: メタ保存で Lesson と content が更新される

- **WHEN** ユーザーがメタを編集して保存する
- **THEN** 対応する `Lesson` のフィールドが更新される
- **AND** `content` のフロントマターが正規化された値で再生成される

### Requirement: Pane3 ヘッダーとモード切替

Pane3 のツールバー（高さ `h-12`）は、左に現在レッスンの日本語名（`lesson` フィールド）を表示しなければならない（SHALL）。モード切替（編集モード・プレビュー・差分）は右寄せで表示しなければならない（SHALL）。レッスンメタ編集の入口はツールバー最右に配置しなければならない（SHALL）。「ファイルを開く」操作を Pane3 に表示してはならない（MUST NOT）。編集モードの表示ラベルは「編集モード」とし、「Markdown」という表記を用いてはならない（MUST NOT）。

#### Scenario: ヘッダーにレッスン名とメタ編集がある

- **WHEN** レッスンが選択され Pane3 が表示される
- **THEN** ツールバー左にレッスン名が表示される
- **AND** ツールバー最右からメタ編集を開ける

#### Scenario: モードタブは右寄せ

- **WHEN** Pane3 ツールバーを表示する
- **THEN** 編集モード・プレビュー・差分のタブ群はレッスン名の右側（右寄せ）に配置される

#### Scenario: ファイルを開くがない

- **WHEN** Pane3 ツールバーを表示する
- **THEN** 「ファイルを開く」ボタンは存在しない

### Requirement: 編集モードの行番号

編集モードでは、本文エディタの左に 1 始まりの行番号を表示しなければならない（SHALL）。行番号は薄いグレーで視覚的に控えめとし、縦スクロールは本文 textarea と同期しなければならない（SHALL）。

#### Scenario: 行番号が本文と同期スクロールする

- **WHEN** ユーザーが編集モードで本文を縦スクロールする
- **THEN** 行番号ガターも同じスクロール位置に追随する

### Requirement: プレビューは本文を GFM で表示する

プレビューモードでは、フロントマターをレンダリングしてはならない（MUST NOT）。本文のみを `react-markdown` でレンダリングし、GitHub Flavored Markdown（タスクリスト等）を有効にしなければならない（SHALL）。Typography は読みやすい資料向けのスタイル（例: `prose`）を用いなければならない（SHALL）。

#### Scenario: プレビューにフロントマターが出ない

- **WHEN** ユーザーがプレビューモードを開く
- **THEN** 画面上に YAML フロントマターのキー・値行は表示されない
- **AND** 本文の見出し・段落が表示される

### Requirement: 差分モードは content 全文を表示する

差分モードでは、当該レッスンの `content` 文字列全体（フロントマターと本文）に対する git 差分を表示しなければならない（SHALL）。

#### Scenario: フロントマター変更が差分に含まれる

- **WHEN** ユーザーがメタのみを変更して差分モードを開く
- **THEN** 差分出力にフロントマター行の変更が含まれる

### Requirement: 確認問題の記法

レッスン本文内の確認問題は、独自の directive 構文（例: `:::quiz`）を用いてはならない（MUST NOT）。確認問題は、見出し `### 確認問題` の直後に、GFM タスクリスト（`- [ ]` / `- [x]`）で記述しなければならない（SHALL）。見出し文言は `### 確認問題` に固定する（SHALL）。

#### Scenario: 確認問題は固定見出しとタスクリスト

- **WHEN** 作者が確認問題を本文に含める
- **THEN** `### 確認問題` が使われる
- **AND** 選択肢は `- [ ]` または `- [x]` のリストである

#### Scenario: プレビューでタスクリストが表示される

- **WHEN** 本文に `### 確認問題` とタスクリストがあるレッスンをプレビューする
- **THEN** 見出しとチェックボックス付きリストがレンダリングされる

### Requirement: レッスンステータスの語彙

レッスンのステータスは `open` / `in_progress` / `done` のいずれかでなければならない（SHALL）。 Pane3 メタモーダルおよび `Lesson.status` はこの語彙を用いなければならない（SHALL）。`draft` を新規データに保存してはならない（MUST NOT）。

#### Scenario: 新規レッスンの既定ステータス

- **WHEN** 新規レッスンが作成される
- **THEN** ステータスは `open` である
