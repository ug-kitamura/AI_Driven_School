# pane3-agent-view Specification

## Purpose

DX Training Studio の Pane 3 Agent ビューの UI と操作を定義する。AI チャット、スキル呼び出し（`/`）、ファイル参照（`@`）、草稿のエディタ挿入、ストリーミング停止などのユーザー向け挙動を規定する。
## Requirements
### Requirement: Agent ビュー切り替え
Pane 3 のモード切り替え UI に Agent ビューボタンが存在しなければならない（SHALL）。ボタンは差分ビューボタンの右隣に配置されなければならない（SHALL）。

#### Scenario: Agent ビューに切り替える
- **WHEN** ユーザーが Pane 3 の Agent ビューボタンをクリックする
- **THEN** Pane 3 が AI チャット画面に切り替わる

#### Scenario: 他モードから Agent ビューに戻れる
- **WHEN** Agent ビュー表示中にユーザーが編集・プレビュー・差分のいずれかのボタンをクリックする
- **THEN** 選択したモードに切り替わる

### Requirement: AI チャット UI
Agent ビューにはメッセージ入力欄と送信ボタンを備えた AI チャット UI が表示されなければならない（SHALL）。ユーザーメッセージと AI 応答が時系列で表示されなければならない（SHALL）。ユーザーメッセージは右寄せの吹き出しスタイルで表示されなければならない（SHALL）。AI 応答は左寄せで表示されなければならない（SHALL）。送信ボタンは上矢印アイコンのみのボタンでなければならない（SHALL）。ストリーミング中は入力欄右下に停止ボタンが表示されなければならない（SHALL）。入力欄左下には **保存済みワークスペース設定の AI モデル** の表示名が表示されなければならない（SHALL）。表示名は `resolveModelLabel(settings.aiModel)` により slug から決定し、ハードコードしてはならない（MUST NOT）。

#### Scenario: メッセージを送信する
- **WHEN** ユーザーがテキストを入力して送信ボタンをクリックする
- **THEN** ユーザーメッセージがチャット履歴に追加され、AI 応答の取得が開始される

#### Scenario: ストリーミング応答を表示する
- **WHEN** AI 応答がストリーミングで返される
- **THEN** 応答テキストが逐次チャット画面に表示される

#### Scenario: ユーザーメッセージを右寄せ吹き出しで表示する
- **WHEN** ユーザーがメッセージを送信する
- **THEN** チャット履歴上で当該メッセージが右寄せの吹き出しとして表示される

#### Scenario: ストリーミング中に停止する
- **WHEN** ユーザーがストリーミング中に停止ボタンをクリックする
- **THEN** 部分応答と直前のユーザーメッセージが履歴から削除され、ユーザーメッセージの内容が入力欄に復元される

#### Scenario: 保存済みモデル名を表示する
- **WHEN** Agent ビューが表示される
- **AND** ワークスペース設定に `aiModel: claude-sonnet-4-6` が保存されている
- **THEN** 入力欄左下に `Claude Sonnet 4.6` が表示される

#### Scenario: 設定保存後にフッター表示が更新される
- **WHEN** ユーザーが設定ダイアログで AI モデルを変更して保存する（Claude Sonnet 4.6 のみ保存可能）
- **AND** Agent ビューを表示している
- **THEN** 入力欄左下のモデル表示名が保存後の `aiModel` に対応するラベルに更新される

### Requirement: スラッシュコマンドによるスキル呼び出し
チャット入力欄で `/` を入力すると、`.claude/skills/` 配下のスキルが skill id のアルファベット順でオートコンプリート表示されなければならない（SHALL）。`/ ` の後に文字を入力した場合は skill id、スキル名（name）、description のいずれかにその文字列が部分一致するスキルだけを表示しなければならない（SHALL）。候補はスキル名を主表示とし、直後に description をうっすら表示しなければならない（SHALL）。候補選択後、アクティブスキルが設定され、送信時にそのスキルで invoke が実行されなければならない（SHALL）。`/` メニューには組み込みコマンド `clear` と `export` も表示されなければならない（SHALL）。

#### Scenario: `/` でスキル候補をアルファベット順表示する
- **WHEN** ユーザーがチャット入力欄で `/` を入力する
- **THEN** skill id がアルファベット順のスキル候補リストが表示される

#### Scenario: スキル候補の表示形式
- **WHEN** `create-draft` スキルが候補に含まれる
- **THEN** 主表示は frontmatter の `name`（スキル名）で、その直後にうっすら `description` が表示される

#### Scenario: スキル候補を部分一致でフィルタする
- **WHEN** ユーザーが `/draft` のように `/` の後に文字を入力する
- **THEN** skill id、name、description のいずれかに `draft` が部分一致するスキルのみが候補に残る

#### Scenario: スキルを選択して実行する
- **WHEN** ユーザーが候補から `create-draft` を選択し、メッセージを送信する
- **THEN** `create-draft` スキルで invoke が実行される

#### Scenario: スキル未選択で送信する
- **WHEN** アクティブスキルが未設定の状態で送信する
- **THEN** エラーメッセージが表示され、`/` によるスキル選択が促される

#### Scenario: clear コマンドを実行する
- **WHEN** ユーザーが `/` メニューから `clear` を選択し確認する
- **THEN** 現在のチャットセッションが削除される

#### Scenario: export コマンドを実行する
- **WHEN** ユーザーが `/` メニューから `export` を選択する
- **THEN** 現在セッションの会話が Markdown ファイルとしてダウンロードされる

### Requirement: @ によるファイル参照
チャット入力欄で `@` を入力すると、`contents/**/*.md` の全ファイルがオートコンプリート表示されなければならない（SHALL）。選択中レッスンがある場合はそのファイルを先頭に、残りは path のアルファベット順で表示しなければならない（SHALL）。`@` の後に文字を入力した場合は、path または file name の部分一致でフィルタしなければならない（SHALL）。候補はファイル名を主表示とし、直後にワークスペースルートからの相対パスをうっすら表示しなければならない（SHALL）。

#### Scenario: `@` で全ファイルを表示する
- **WHEN** ユーザーがチャット入力欄で `@` を入力する（フィルタ文字なし）
- **THEN** `contents/**/*.md` の全ファイルが表示され、選択中レッスンがある場合はそのファイルが先頭である

#### Scenario: 候補の表示形式
- **WHEN** `@contents/series/course/lesson.md` が候補に含まれる
- **THEN** 主表示は `lesson.md` で、その直後にうっすら `contents/series/course/lesson.md` が表示される

#### Scenario: ファイル候補を部分一致でフィルタする
- **WHEN** ユーザーが `@lesson` のように `@` の後に文字を入力する
- **THEN** path または file name に `lesson` が部分一致するファイルのみが候補に残る

#### Scenario: ファイルを選択して参照トークンを挿入する
- **WHEN** ユーザーが候補から `contents/foo/bar/lesson.md` を選択する
- **THEN** 入力欄に `@contents/foo/bar/lesson.md` 形式のトークンが挿入される

#### Scenario: 参照ファイル付きメッセージを送信する
- **WHEN** ユーザーが `@contents/foo/bar/lesson.md` を含むメッセージを送信する
- **THEN** 該当ファイルの内容が invoke リクエストに添付され、AI 応答の生成に利用される

#### Scenario: 参照トークンの表示
- **WHEN** ユーザーメッセージに `@path` トークンが含まれる
- **THEN** チャット履歴上で参照ファイルが識別可能なチップまたはスタイルで表示される

### Requirement: 草稿のエディタ挿入

AI 応答に対して **「エディタに上書き」** アクションを提供しなければならない（SHALL）。操作は応答テキストから **markdown コードブロック**（```markdown ... ``` または ``` ... ```）を抽出し、存在すればその**内側の本文**（開始行の ` ```markdown ` / ` ``` ` および対応する閉じフェンス行を除く）を、なければ応答全文を選択中レッスンの編集内容で **置換** しなければならない（SHALL）。抽出結果にフェンス行自体を含めてはならない（MUST NOT）。本文中にネストしたコードブロック（例: ` ```bash `）がある場合、外枠 markdown ブロックの閉じフェンスまで欠落なく抽出しなければならない（SHALL）。4 バッククォート外枠（` ````markdown ` で始まる形式）で包まれた応答でも、内側の ` ```markdown ` ブロック本文を正しく抽出しなければならない（SHALL）。

置換前に、抽出 markdown を `lib/lesson-frontmatter.ts` のレッスンメタ正規化（`normalizeLessonMeta` 相当）で処理し、フロントマターの `series` / `course` / `lesson` を選択中レッスンの位置に合わせ、`status` を `open` | `in_progress` | `done` のいずれかに、`tags` を `[a-z0-9-]+` パターンに適合する値のみに矯正しなければならない（SHALL）。`status: draft` 等の legacy 値は `open` にマップされなければならない（SHALL）。上書き前に確認ダイアログを表示しなければならない（SHALL）。上書き成功後、Pane 3 を **編集ビュー** に自動切替しなければならない（SHALL）。「エディタに上書き」と「コピー」は AI 応答のストリーミングが完了した後にのみ表示されなければならない（SHALL）。

#### Scenario: markdown ブロックを抽出して上書きする

- **WHEN** AI 応答に ```markdown で囲まれた草稿が含まれる

- **AND** ユーザーが「エディタに上書き」をクリックし確認する

- **THEN** コードブロック内のテキスト（フェンス行を除く）がレッスン本文を置換する

- **AND** 置換後の本文に ` ```markdown ` や外枠を閉じる ` ``` ` 行が含まれない

- **AND** Pane 3 が編集ビューになる

#### Scenario: 4 バッククォート外枠付き応答から抽出する

- **WHEN** AI 応答が ` ````markdown ` で始まり、その内側に ` ```markdown ` ブロックがある

- **AND** ユーザーが「エディタに上書き」をクリックし確認する

- **THEN** 内側 markdown ブロックの本文（YAML frontmatter と markdown 本文）のみがレッスン本文を置換する

- **AND** 外枠・内枠のフェンス行は含まれない

#### Scenario: 本文中コードブロックを含む草稿を完全抽出する

- **WHEN** AI 応答の markdown ブロック内に ` ```bash ` 等のネストしたコードブロックが含まれる

- **AND** ユーザーが「エディタに上書き」をクリックし確認する

- **THEN** 外枠 markdown ブロックの閉じフェンス手前までの全文が抽出される

- **AND** ネストしたコードブロックは本文の一部として保持される

#### Scenario: 不正 status を矯正して上書きする

- **WHEN** AI 草稿の frontmatter に `status: draft` が含まれる

- **AND** ユーザーが「エディタに上書き」をクリックし確認する

- **THEN** エディタに書き込まれる frontmatter の status は `open` である

#### Scenario: 不正 tags を除去して上書きする

- **WHEN** AI 草稿の frontmatter に `tags: [git, ブランチ]` が含まれる

- **AND** 選択中レッスンの tags が `[git, branch]` である

- **AND** ユーザーが「エディタに上書き」をクリックし確認する

- **THEN** エディタに書き込まれる tags は `[git, branch]` である（日本語 tag は除去され、fallback の合法 tag が保持される）

#### Scenario: レッスン未選択時の上書き

- **WHEN** レッスンが選択されていない状態で「エディタに上書き」をクリックする

- **THEN** 操作は無効化されるか、エラーメッセージが表示される

#### Scenario: ストリーミング中は上書きボタンを非表示にする

- **WHEN** AI 応答がストリーミング中である

- **THEN** 当該応答に「エディタに上書き」ボタンは表示されない

### Requirement: チャット送信後のスクロールとフォーカス

Agent ビューは、ユーザーがメッセージを送信した後、チャット履歴スクロールを **最新メッセージ付近（末尾）** に維持しなければならない（SHALL）。同時に入力欄（textarea）へフォーカスを戻さなければならない（SHALL）。ストリーミング中は新しい delta 到着時も末尾追従しなければならない（SHALL）。ユーザーが履歴を手動で上方スクロールしている間は自動スクロールを抑制してよい（MAY）。

#### Scenario: 送信後に末尾と入力欄が維持される

- **WHEN** ユーザーがメッセージを送信する
- **THEN** チャット履歴は最新メッセージが見える位置にスクロールされる
- **AND** 入力欄にフォーカスが残る

#### Scenario: ストリーミング中に末尾追従する

- **WHEN** AI 応答がストリーミング中である
- **AND** ユーザーが履歴末尾付近を表示している
- **THEN** 新しいテキスト到着ごとに末尾追従する

### Requirement: ビュー切替中も Agent 処理を継続する

Pane 3 の編集・プレビュー・差分ビューへ切り替えても、`AgentChatPane` を unmount してはならない（MUST NOT）。進行中の AI invoke / ストリーミングは裏で継続しなければならない（SHALL）。Agent ビューに戻ったとき、進行中または完了済みのメッセージ状態が反映されていなければならない（SHALL）。

#### Scenario: 編集ビュー切替中もストリーミング継続

- **WHEN** ユーザーが Agent ビューでメッセージを送信し AI がストリーミング中である
- **AND** ユーザーが編集ビューに切り替える
- **THEN** ストリーミングは中断されない
- **AND** Agent ビューに戻ると応答が更新されている

### Requirement: ストリーミング中のレッスン切替確認

AI 応答がストリーミング中にユーザーがレッスンまたはコースの選択を変更しようとした場合、確認 `AlertDialog` を表示しなければならない（SHALL）。ダイアログには、切替により応答が中断される旨と、途中までの内容が保存される旨を示さなければならない（SHALL）。「キャンセル」選択時は選択を変更してはならない（MUST NOT）。「切り替える」選択時は進行中 invoke を abort し、部分 assistant 応答を保持したうえで選択変更を完了しなければならない（SHALL）。

#### Scenario: ストリーミング中にレッスン切替をキャンセルする

- **WHEN** AI がストリーミング中である
- **AND** ユーザーが別レッスンを選択し確認ダイアログで「キャンセル」を選ぶ
- **THEN** 選択レッスンは変わらない
- **AND** ストリーミングは継続する

#### Scenario: ストリーミング中にレッスン切替を確認する

- **WHEN** AI がストリーミング中である
- **AND** ユーザーが別レッスンを選択し確認ダイアログで「切り替える」を選ぶ
- **THEN** invoke が abort される
- **AND** 切替元レッスンの部分 assistant 応答が履歴に残る
- **AND** 切替先レッスンの会話が表示される

### Requirement: 部分応答からの続きを生成

最後の assistant メッセージが空でなく、かつその直後に follow-up がない状態（invoke 中断直後等）では、当該 assistant メッセージに「続きを生成」操作を表示しなければならない（SHALL）。操作実行時、アクティブスキルを維持し、user メッセージ「つづきよろしく」を履歴（部分 assistant 含む）に追加して invoke しなければならない（SHALL）。

#### Scenario: 続きを生成する

- **WHEN** assistant メッセージが途中まで出力された状態で「続きを生成」をクリックする
- **THEN** user メッセージ「つづきよろしく」が送信される
- **AND** 部分 assistant を含む履歴をもとに AI 応答が開始される

#### Scenario: 完了済み assistant には続きを生成を表示しない

- **WHEN** 最後の assistant メッセージが invoke 完了済みである
- **THEN** 「続きを生成」操作は表示されない

### Requirement: tool call 折りたたみ UI

Agent ビューは、invoke 中に実行された tool call を折りたたみ可能なブロックとして表示できなければならない（SHALL）。表示テキストは最小限（caveman スタイル）とし、冗長な説明文を含めてはならない（MUST NOT）。例: `🔍 search: ブランチ → 3件`、`✓ select: 1,3`。

#### Scenario: 検索 tool を折りたたみ表示する

- **WHEN** `search_company_context` が実行される

- **THEN** チャット履歴に折りたたみ可能な tool ブロックが表示される

- **AND** 展開時に query と hit 件数が確認できる

#### Scenario: デフォルトは折りたたみ

- **WHEN** tool ブロックが表示される

- **THEN** 初期状態は折りたたまれている
