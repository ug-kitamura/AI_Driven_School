# Git のインストール手順

## 学習目標

このレッスンを終えると、以下ができるようになります。

- 自分の OS（Windows / Mac）に Git をインストールできる
- ターミナルで `git --version` を実行してインストールを確認できる
- 社内環境での申請・インストール手順を理解している

---

## はじめに：社内環境での注意事項

> **社内ワンポイント**
>
> 社内では **個人での直接ダウンロードは不可** です（Windows・Mac いずれも）。
> 申請からインストール完了まで数日かかる場合があるため、**研修開始の 1 週間前には申請を出すことを推奨** します。

---

## Windows（社内環境）

社内の承認・配布システムを通じてインストールします。

### 手順

1. **社内独自の承認システム**から Git のインストール申請を出す
   - 同時に VS Code も申請しておくと手続きが一度で済みます（詳細は後述）
2. 承認後、**社内のソフトウェア配布ポータル**から Git をインストールする
   - Git のインストーラには **SourceTree が同梱**されており、両方が一緒にインストールされます

### Git 操作ツールの選択

インストール後は、以下のいずれかのツールで Git を操作できます。自分のスタイルに合わせて選んでください。

| ツール | 向いている人 |
|---|---|
| SourceTree | Git 操作に慣れていない方。GUI でコミット履歴・ブランチ状態を確認しやすい |
| VS Code 内蔵 Git 機能 / ターミナル（`git` コマンド） | VS Code に慣れている方 |

## Mac（社内環境）

Mac の場合も、社内の承認・配布システムを通じてインストールします。個人で Homebrew から直接インストールすることは **避けてください**。

> **社内ワンポイント**
>
> 社内 Chocolatey ベースのパッケージ管理システムが利用できる場合があります。所属部署の IT ポリシーに従い、IT ヘルプデスクに確認してください。

### 手順

1. 社内独自の承認システムからインストール申請を出す
2. 承認後、社内のソフトウェア配布ポータルまたは指定の手順に従ってインストールする

---

## インストールの確認

OS を問わず、インストール後は以下のコマンドで動作確認をしてください。

```bash
git --version
```

以下のようにバージョン番号が表示されれば成功です。

```
git version 2.x.x
```

バージョンが表示されない場合は、**IT ヘルプデスクへ問い合わせ**てください。

---

## セットアップ完了チェックリスト

Git のインストールが完了したら、以下を自己チェックしてください。

- [ ] `git --version` でバージョンが表示される
- [ ] SourceTree または VS Code から Git が使える状態になっている
- [ ] Git リポジトリのクローンができる
