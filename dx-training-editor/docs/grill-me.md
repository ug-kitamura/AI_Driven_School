# DX Training Editor — グリルセッション記録

**実施日**: 2026-05-16  
**目的**: 4ペインワークスペースUIの仕様を詰める  
**ステータス**: 全未決事項 解決済み ✅（ゲーミフィケーションのみ後回し）

---

## 背景・解くべき課題

- 社内新規プロジェクトでツール環境が大きく変化（特にGit・GitHubは不慣れな者が多い）
- 対象者数に対してトレーニングできる人が限られている → **独習形式**が必要
- トレーナーの負担を減らし、受講者が自分のペースでレベルに合った内容を進められるようにする

### DXトレーニングの設計方針

1. **基本独習形式** — トレーナー不要で自己完結
2. **マイクロトレーニング** — 各レッスン10〜20分
3. **ゲーミフィケーション** — レベルアップ構造で受講者のモチベーションを維持（詳細は後回し）
4. **DXトレーニング曼陀羅** — コース修了で次の複数コースが解放、または複数コース受講済みでないと次に進めないフローチャート構造。最終目標は「DX免許皆伝」

### DXトレーニング曼陀羅とエディタの関係

- **曼陀羅の構築・可視化は別ツールが担当**（エディタのスコープ外）
- DX Training Editor は各コースに「受講対象者」「前提コース」「次のコース」のメタ情報を持ち、曼陀羅ツールがその情報を読み取る

## 認識がそろった箇所

### アプリ概要

| 項目 | 内容 |
|---|---|
| アプリ名 | DX Training Editor |
| ディレクトリ名 | `dx-training-editor` |
| 目的 | 社内DXツールトレーニングのコンテンツを作成支援するUI |
| 対象ユーザー | 社内研修担当者1人が使う制作専用ツール（マルチユーザー・コラボ機能は不要） |
| 技術スタック | Next.js + TypeScript + Tailwind CSS + shadcn/ui（workspace-ui-kit と同一） |
| 制作スタイル | AIで骨子を生成 → このツールで読み込み・編集 |
| git連携 | git管理下のディレクトリで使用する前提。commit・pushはスコープ外 |

### コンテンツ構造

- **階層**: シリーズ → コース → レッスン（3層）
- **対象ツール**: Git・GitHub（スタート）→ Python・CI/CD（GitHub Actions / Jenkins）・AI活用へ順次拡張

#### コースメタ情報（レッスンYAMLとは別に管理）

各コースは以下のメタ情報を持つ：

| フィールド | 内容 |
|---|---|
| `id` | コースの一意ID |
| `name` | コース名 |
| `target_audience` | 想定する受講対象者（テキスト） |
| `prerequisites` | **別シリーズ**の、前もって受講すべきコース ID（0件以上）。同シリーズ内の前後は `series.courses[]` の並びで決まる |
| `next_courses` | **別シリーズ**の、修了後に受講できるコース ID（0件以上）。同シリーズ内の次は並び順で決まる |
| `lessons` | 配下のレッスン一覧 |

### 4ペイン構成

| ペイン | 役割 | 主な機能 |
|---|---|---|
| **Pane 1** | シリーズ＋コース一覧 | グローバル進捗バー・シリーズ折りたたみ・コースステータスバッジ、**折りたたみ可** |
| **Pane 2** | コースメタ情報＋レッスン一覧 | 上部：コース名・対象・ミニ曼陀羅（編集ダイアログでメタ変更）、下部：コース進捗バー・レッスンステータスバッジ・CRUD |
| **Pane 3** | マークダウンエディタ | 3モード切り替え（下記参照）、ファイルピッカー・テキストペーストで読み込み、新規テンプレート作成 |
| **Pane 4** | 画像アセットマネージャー | 4タブ構成（下記参照）、**折りたたみ可** |

#### Pane 1 の表示構造

```
[全シリーズ統合 グローバル進捗バー ████░░ 8/15]
▼ Git完全マスターシリーズ         ████░░ 3/5 コース完成
    ○ Git概念マスターコース    [完成]
    ○ Git環境構築コース        [作成中]
    ○ Gitブランチ操作コース    [未着手]
▶ Python基礎シリーズ              ░░░░░░ 0/3 コース完成
```

- Pane1 最上部：全シリーズを合算したグローバル進捗バー
- シリーズ：大見出し、折りたたみ/展開可、シリーズ内コースの進捗バー表示
- コース：シリーズ配下の小見出し、コース単位のステータスバッジ表示

#### Pane 2 の表示構造

```
┌─────────────────────────────────┐
│ 対象者  : Git未経験の開発者       │
│ 前提    : [Git概念] [Git環境]    │  ← クリックでそのコースにジャンプ
│ 次のコース: [Gitブランチ] [GitHub] │  ← クリックでそのコースにジャンプ
│                         [編集]   │
│                                  │
│  [Git概念] ──→ [★本コース★]     │  ← Mermaidで動的生成
│  [Git環境] ──↗   ↓              │
│          [Gitブランチ][GitHub入門] │
├─────────────────────────────────┤
│ [コース進捗バー ██░░░ 2/5]       │
│  [未着手] バージョン管理ってなに？  │
│  [完成]   Gitのインストール手順    │
│  [作成中] 最初のコミット           │
└─────────────────────────────────┘
```

- Pane2 上部：コースメタ情報テーブル（受講対象者・前提・次コース）＋ 編集ボタン
- コース名クリックでPane1のコース選択がジャンプ
- Mermaidミニグラフ：`前提コース → 現在のコース → 次のコース` を動的生成
- Pane2 下部：コース進捗バー＋レッスン一覧（ステータスバッジ・CRUD）

#### Pane 3 の3モード

| モード | 説明 |
|---|---|
| ① インライン編集（デフォルト） | Notion風。クリックしたブロックだけ編集モード、他はレンダリング済み |
| ② 生マークダウン編集 | 全体をプレーンテキストで編集（AIペースト・一括編集向け） |
| ③ 差分表示 | `git diff HEAD <ファイルパス>` でHEADとの差分を表示（読み取り専用） |

#### Pane 4 の4タブ

| タブ | 動作 |
|---|---|
| アップロード | ドラッグ&ドロップ＋クリップボード貼り付け（**実動作**） |
| 履歴一覧 | 全画像を全体共有で一覧表示、クリックでPane3のカーソル位置に `![ファイル名](パス)` を挿入（**実動作**） |
| AI生成 | タブUIのみ（**モック**） |
| Web検索 | タブUIのみ（**モック**） |

### ステータス定義

**レッスンステータス**（手動設定）:

| 値 | 表示 |
|---|---|
| `open` | 未着手 |
| `in_progress` | 作成中 |
| `done` | 完成 |

**コース・シリーズステータス**（レッスンステータスから自動計算）:

| 条件 | 表示ステータス |
|---|---|
| 配下がすべて「未着手」 | 未着手 |
| 配下がすべて「完成」 | 完成 |
| それ以外（「作成中」または「完成」が1つ以上） | 作成中 |

### YAMLフロントマター

```yaml
series: Git完全マスターシリーズ
course: Git環境構築コース
lesson: Gitのインストール手順
status: open           # open | in_progress | done
description: 簡潔な説明
tags: [git, setup]
estimated_minutes: 0
author: 山田太郎
```

フロントマターは Pane3 のメタ編集モーダルでのみ編集する。編集モードの textarea には本文のみ表示する。`series` / `course` はワークスペース上の親の日本語名が正本であり、保存時にフロントマターへ上書きされる。

新規レッスン作成時は series・course を含むテンプレートが展開される（ステータス既定は `open`）。

### 確認問題の記法

```markdown
### 確認問題

バージョン管理の主な目的は何ですか？

- [ ] ファイルを圧縮する
- [x] 変更履歴を記録する
- [ ] ファイルを暗号化する
```

見出しは `### 確認問題` に固定。本文マークダウン内に記述（AI生成時に本文と一緒に出力できる）。独自 directive（`:::quiz` 等）は使わない。

### データ・永続化

- **モックアップ段階**: ローカルJSON（`data/` 配下）でサンプルデータを管理
- **将来の本番**: GitHubリポジトリにマークダウンファイルとして保存（YAMLフロントマター形式）
- **CRUD**: 追加・削除・並び替えはセッション内メモリのみ（保存なし）

## 解像度が上がった箇所

- **Pane3の役割**: エディタはゼロから書く場所ではなく「AIが生成した原稿を受け取って磨く」場所。3モード（編集モード・プレビュー・差分）はその流れを支える構造。プレビューは `react-markdown` + GFM、本番資料は Python + MkDocs で別変換
- **画像管理の痛み**: 毎回ファイルパスを手打ち・スクショの整理ができていない → Pane4の画像アセットマネージャーで解消
- **進捗把握の痛み**: どのレッスンが穴か分からない → Pane1（シリーズ・コース粒度）＋Pane2（レッスン粒度）の2層で解消
- **ステータス設計の背景**: コース・シリーズを手動設定すると不整合が起きるため自動計算に統一。「作業が始まった＝作成中」という直感的なルールに落ち着いた
- **Cursorとの差別化**: 進捗可視化・画像マネージャー・差分表示の3点が一体化しており、Cursorでは代替できない

| workspace-ui-kit（採用管理） | DX Training Editor |
|---|---|

### コンテンツ制作ワークフロー

```
AIスキル / Python スクリプト
    ↓ マークダウン生成（YAMLフロントマター＋本文＋### 確認問題）
DX Training Editor
    ↓ Pane3 編集モードで本文を磨く（メタはメタ編集モーダル）
    ↓ ① プレビューで読みやすさを確認
    ↓ ② Pane4 で画像をアップロード → カーソル位置に挿入
    ↓ ③ Pane2 または Pane3 でレッスンステータスを更新
    ↓ ④ 差分モードでHEADからの変更を確認（content 全文）
将来: Python スクリプトで MkDocs 用資料へ変換 → GitHub へ .md 保存
```

### マークダウンエディタ（採用）

| 用途 | 採用 |
|---|---|
| Pane3 プレビュー | `react-markdown` + `remark-gfm` + Tailwind `prose` |
| Pane3 編集 | `textarea` + 行番号ガター |
| 本番資料 | Python + MkDocs（ワークスペース外） |
