GitHub Actions Tips

workflow_dispatch でいつでも手動実行できるようにしよう

今回の weekly-report.yml のように定期実行ワークフローを作るとき、workflow_dispatch トリガーを追加しておくと、GitHub UI から手動で即実行できて動作確認が格段に楽になります。

on:
  schedule:
    - cron: '0 8 * * 1'   # 毎週月曜 08:00 UTC
  workflow_dispatch:      # ← これを追加！
    inputs:
      dry_run:
        description: 'Dry run (skip Slack post)'
        type: boolean
        default: false

手動実行時に inputs でパラメータを渡せるので、「Slack には投稿しないテスト実行」なども簡単に実現できます。Cherry の GitHub Actions 対応で早速試してみてください！
