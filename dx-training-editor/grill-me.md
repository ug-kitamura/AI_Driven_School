## 認識がそろった箇所

### アプリ概要
- **アプリ名**: DX Training Editor
- **ディレクトリ名 / スラッグ**: `dx-training-editor`
- **目的**: 社内DXツールトレーニングのコンテンツを作成支援するUI
- **対象ユーザー**: 社内研修担当者1人が使う制作専用ツール（マルチユーザー・コラボ機能は不要）
- **技術スタック**: Next.js + TypeScript + Tailwind CSS + shadcn/ui（workspace-ui-kit と同一）
- **制作スタイル**: AIで骨子を生成 → このツールで読み込み・編集

### コンテンツ構造
- **階層**: シリーズ → コース → レッスン（3層）
- **シリーズ**: Pane1の見出しとして表示（ナビゲーション階層には含まない）
- **対象ツール**: Git・GitHub（スタート）→ Python・CI/CD（GitHub Actions / Jenkins）・AI活用へ順次拡張

### 4ペイン構成

| ペイン | 役割 | 主な機能 |
|---|---|---|
| **Pane 1** | シリーズ見出し＋コース一覧 | コース単位の進捗バー（X / Y レッスン完成）、**折りたたみ可** |
| **Pane 2** | レッスン一覧 | ステータスバッジ（未着手・作成中・完成）、CRUD（追加・削除・並び替え） |
| **Pane 3** | マークダウンエディタ | 3モード切り替え：①Notion風インライン編集（デフォルト）②生マークダウン全体編集③差分表示（HEAD比較）、新規テンプレート作成・ファイルピッカー・テキストペーストで読み込み |
| **Pane 4** | 画像アセットマネージャー | 4タブ構成（下記参照）、**折りたたみ可** |

#### Pane 4 — 画像アセットマネージャーのタブ

| タブ | 動作 |
|---|---|
| アップロード | ドラッグ&ドロップ ＋ クリップボード貼り付け（**実動作**） |
| 履歴一覧 | 過去に使用した全画像を一元表示、クリックで Pane 3 に挿入（**実動作**・全体共有スコープ） |
| AI生成 | タブUIのみ（**モック**） |
| Web検索 | タブUIのみ（**モック**） |

### YAMLフロントマター（レッスンのメタデータ）

```yaml
series: Git完全マスターシリーズ
course: Git環境構築コース
lesson: Gitのインストール手順
status: draft          # draft | in_progress | done
description: 簡潔な説明
tags: [git, setup]
estimated_minutes: 15
author: 山田太郎
```

### レッスンステータス
- **3段階**: `未着手`（draft）/ `作成中`（in_progress）/ `完成`（done）
- Pane 1 の進捗バーは `done` のレッスン数 / 総レッスン数で計算

### データ・永続化
- **モックアップ段階**: ローカルJSON（`data/` 配下）でサンプルデータを管理
- **将来の本番**: GitHubリポジトリにマークダウンファイルとして保存（YAMLフロントマター形式）
- **CRUD**: 追加・削除・並び替えはセッション内メモリのみ（保存なし）

### Cursorとの差別化ポイント
1. コース・レッスン単位の進捗バッジ・バーで現在地が一目でわかる
2. 画像アセットマネージャーでスクショを2アクションで本文に挿入できる
3. YAMLフロントマターがレッスンの所属（シリーズ・コース）を自己記述する
4. 新規レッスン作成時に階層情報・著者が自動入力されるテンプレートが出る

## 解像度が上がった箇所

- **Pane 3 の編集体験**: マークダウンをAIで生成し外部から持ち込む前提。エディタはゼロから書く場所ではなく「受け取って磨く」場所
- **画像管理の痛み**: 毎回ファイルパスを手打ちする・スクショの整理ができていない → Pane 4 で解消
- **進捗把握の痛み**: どのレッスンが穴か分からない → Pane 1 + Pane 2 の2層バッジで解消
- **画像スコープ**: 履歴は全体共有（シリーズ・コースをまたいで再利用可）

## 追加で決まった事（第2セッション）

- **Pane3 編集UX**: Notion風インライン編集（デフォルト）＋ トグルで生マークダウン全体編集モードに切り替え可
- **画像挿入形式**: Pane4で画像クリック → Pane3のカーソル位置に `![ファイル名](パス)` を挿入
- **ファイル読み込み**: ファイルピッカー（OSダイアログ）＋ テキストエリアへの直接ペーストの両対応
- **ペイン折りたたみ**: Pane1・Pane4 の両方を折りたたみ可（Pane3 エディタ集中モード対応）
- **確認問題の記法**: 本文マークダウン内に `:::quiz` ブロック記法で記述（AI生成時に一緒に出力できる）
- **カラースキーム**: 青・白・グレー基調、`custom-colors.xlsx` のパレットを使用（下記参照）

## 追加仕様（第4セッション）

### Git差分表示（Pane3）

- このツールはgit管理下のディレクトリで使用することを前提とする
- git commit・push はこのツールのスコープ外
- Pane3に**差分表示モード**を追加（3つ目のモード）
- 比較対象：**HEAD（最後のコミット）** との差分
- 実装：Next.js APIルートから `git diff HEAD <ファイルパス>` を呼び出す

**Pane3 の3モード:**

| モード | 説明 |
|---|---|
| ① インライン編集（デフォルト） | Notion風、クリックしたブロックを編集 |
| ② 生マークダウン編集 | 全体をプレーンテキストで編集（AIペースト向け） |
| ③ 差分表示 | HEADとの差分をdiff形式で表示（読み取り専用） |

## 追加仕様（第3セッション）

### Pane1 の構造

```
[全シリーズ統合ステータスバー]
▼ Git完全マスターシリーズ  ████░░ 3/5
    ○ Git概念マスターコース    [完成]
    ○ Git環境構築コース        [作成中]
▶ Python基礎シリーズ       ░░░░░░ 0/3
```

- シリーズ：大見出し、折りたたみ/展開可、シリーズ内コース数の進捗バー表示
- コース：シリーズ配下の小見出し、コース単位のステータスバッジ表示
- Pane1 最上部：全シリーズを合算したグローバル進捗バー

### Pane2 の構造

```
[コース全体ステータスバー]  ██░░░ 2/5 レッスン完成
  [未着手] バージョン管理ってなに？
  [完成]   Gitのインストール手順
  [作成中] 最初のコミット
```

- Pane2 最上部：選択中コースのレッスン進捗バー
- 各レッスン：ステータスバッジ（未着手・作成中・完成）

### ステータス自動計算ルール（コース・シリーズ共通）

| 条件 | 表示ステータス |
|---|---|
| 配下がすべて「未着手」 | 未着手 |
| 配下がすべて「完成」 | 完成 |
| それ以外（「作成中」または「完成」が1つ以上） | 作成中 |

コース・シリーズのステータスはレッスンステータスから自動計算（手動設定なし）。
