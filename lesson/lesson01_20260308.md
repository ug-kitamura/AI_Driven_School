        ADS 憲章（全10個）
            1. 90点 -> 100点の戦いをする
                最後の仕上げに最も時間と労力が必要
                最後の10点をつめきるツール作成に注力する
                上流よりも下流から
            2. 理解できないものを作らない
                最後のツメができなくなるから
                すべてを自動化しようとしたり、AI に丸投げは大抵失敗する
            3. 自分の武器を自分で作る
                作りながら理解すれば良い
            4. 図解にする
                自分用と共有用がある
                Claude > Gemini > GPT の順に図解に向いている
            5. 議論する前にプロトタイプを作る
                あれこれ議論するより、さっさとモックアップを作る方が仕事が前に進む
            ※ 残り5個は次回以降の予定

    今日の講義内容
        ツール紹介
            1. 図解フィードバックツール
                Skill とは？
                surge という web ページにアップロード
                コメント書くこと可能
                ※ 非エンジニアが作成
            2. デイリーレポートツール
                タスク確認
                進捗確認
                やることレポート
                ※ 非エンジニアが作成
            3. 動画編集ツール
                多くの行程、多くのツールを１画面に集約
                ブラウザ上で完結
                ※ 非エンジニアが作成
            4. クイズ作成ツール
                シリーズ -> コース -> レッスン構成
                    全2万問
                    全イラスト付き
                human in the loop 体制で制作
                スキル x サブエージェント x フックで作成？
                ※ 真子さんが作成
        コンテンツ作成
            コンテンツビューアーを開発
            講義内容も AI に作らせている
            スライドと原稿を同じページで管理
            カーソルと連携
            ツール + コンテンツ = スキルと定義し、最高のクォリティを追及
        仕事の変化とマインドセット
            ツールを自分で作る
                頭に思い浮かんだことを AI を使って形にすることが重要
                ツールを自分で作ることで圧倒的な効率化ができる
            AI による需要の変化
                テクノロジーの進化により、コストが下がり、需要がむしろ増えていく
                需要自体はなくならず、さらに複雑化・高度化していく
                翻訳の例
                    翻訳は文化・文脈まで考慮したものが出来るようになる
                    ただ翻訳しただけだと価値はなく、さらにその先を求められるようになる
            今後重視されること
                全く同じことをしていてはダメ
                顧客体験の設計、アーキテクチャの設計が重要になる
                プロトタイプを受取り -> 仕組みを理解し -> アレンジするサイクルを重視する
            『ただ使える』から『何がしたいか』へ
                目的意識を持つことが重要で、そのためには仕組みの理解が必須
                基礎的な技術を理解することで、目的意識を持ちやすくなる
                テクノロジーだけではだめで、実際に作ってみて理解することが大切
        ワーク
            1. AI を使った事例
                会社の議事録を社内 AI システム（Gemini 2.5） で作った
                皆からすごいねーと言われたが
                丁度いい感じに作れず、結構手直しが必要だった
                もっと良くする余地がある
            2. アプリ作成の進め方
                まずは要件定義をする
                    何人で使う？
                    ステータス遷移は？
                    カレンダー機能は必要？
                web アプリの作り方を聞いて
                スケジュール決め
                AI に聞きながらやると良いと思うが、もはや自動で作れるのでは？
            3. やめること宣言
                youtube をだらだら見るのをやめます
                ネフライトさんのフォートナイト実況とか
                頭使わなくていいから疲れた時についついダラダラ見てしまう
            4. 面倒なこととその正体
                git のトレーニングをやってくれと言われている
                git の膨大な情報から丁度自分たちに今必要な部分を抽出するのが面倒
                さらにそれを良い感じの資料にするのが面倒
                正体は情報収集・取捨選択・見やすく加工の大変さ
                -> まさに AI で楽できそう
            5. テクノロジーにより簡単になったこと
                エクセルのデータをマクロや関数であれこれするの面倒だし管理大変
                python という学習コストの低いプログラミング言語で代替出来るようになった
                今後はさらに python プログラミングも AI にやらせるようなる（既になっている）
