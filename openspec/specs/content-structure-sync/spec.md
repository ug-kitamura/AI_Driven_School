## ADDED Requirements

### Requirement: レッスン追加時のファイル生成
UI でレッスンを追加した場合、コースフォルダ内に適切な数値プレフィックスを持つ `.md` ファイルを生成しなければならない（SHALL）。ファイルにはフロントマターとデフォルト本文を含めなければならない（SHALL）。

#### Scenario: レッスンを末尾に追加する
- **WHEN** ユーザーが UI で既存コースにレッスンを追加し、レッスン名を入力する
- **THEN** `contents/<シリーズ>/<コース>/<N+1>_<レッスン名>.md` が生成され、フロントマターとデフォルト本文が書き込まれている

### Requirement: コース追加時のフォルダ生成
UI でコースを追加した場合、シリーズフォルダ内に適切な数値プレフィックスを持つコースフォルダと空の `_course.json` を生成しなければならない（SHALL）。

#### Scenario: コースを末尾に追加する
- **WHEN** ユーザーが UI でシリーズにコースを追加し、コース名を入力する
- **THEN** `contents/<シリーズ>/<N+1>_<コース名>/` フォルダと `_course.json` が生成されている

### Requirement: シリーズ追加時のフォルダ生成
UI でシリーズを追加した場合、`contents/` 直下にシリーズフォルダを生成し、`_series-order.json` を更新しなければならない（SHALL）。

#### Scenario: シリーズを追加する
- **WHEN** ユーザーが UI でシリーズを追加し、シリーズ名を入力する
- **THEN** `contents/<シリーズ名>/` フォルダが生成され、`_series-order.json` の末尾にシリーズ名が追加されている

### Requirement: リネーム時のフォルダ/ファイル名変更
UI でシリーズ・コース・レッスンの名前を変更した場合、対応するフォルダ名またはファイル名を変更しなければならない（SHALL）。数値プレフィックスは保持されなければならない（SHALL）。

#### Scenario: コースをリネームする
- **WHEN** ユーザーが UI でコース名を `旧コース名` から `新コース名` に変更する
- **THEN** `01_旧コース名/` フォルダが `01_新コース名/` にリネームされている

#### Scenario: レッスンをリネームする
- **WHEN** ユーザーが UI でレッスン名を `旧レッスン名` から `新レッスン名` に変更する
- **THEN** `02_旧レッスン名.md` が `02_新レッスン名.md` にリネームされ、フロントマターの `lesson:` フィールドも更新されている

### Requirement: 並び替え時の数値プレフィックス振り直し
UI でコースまたはレッスンの表示順を変更した場合、同一階層内のすべてのフォルダ/ファイルの数値プレフィックスを振り直さなければならない（SHALL）。

#### Scenario: レッスンを上位に移動する
- **WHEN** ユーザーが UI で `03_レッスンC` を 1番目に移動する
- **THEN** ファイル名が `01_レッスンC`, `02_旧01_レッスンA`, `03_旧02_レッスンB` にリネームされる

#### Scenario: 並び替え中にエラーが発生する
- **WHEN** リネーム操作中にサーバーエラーが発生する
- **THEN** ロールバックを試み、UI にエラートーストを表示する。state は変更前に戻す

### Requirement: 削除時のフォルダ/ファイル削除
UI でシリーズ・コース・レッスンを削除した場合、対応するフォルダまたはファイルを削除しなければならない（SHALL）。コース削除時は配下のすべてのレッスン `.md` ファイルも削除されなければならない（SHALL）。

#### Scenario: レッスンを削除する
- **WHEN** ユーザーが UI でレッスンを削除する
- **THEN** 対応する `.md` ファイルが削除され、同一コース内の残りレッスンの数値プレフィックスが振り直される

### Requirement: ファイルシステム禁止文字のサニタイズ
シリーズ・コース・レッスン名にファイルシステム禁止文字（`/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`）が含まれる場合、`_` に置換してファイル名に使用しなければならない（SHALL）。UI に表示される名前はそのまま保持されなければならない（SHALL）。

#### Scenario: 禁止文字を含む名前でレッスンを作成する
- **WHEN** ユーザーが `レッスン/テスト` というレッスン名を入力する
- **THEN** ファイル名は `01_レッスン_テスト.md` となり、フロントマターの `lesson:` には `レッスン/テスト` が保存される
