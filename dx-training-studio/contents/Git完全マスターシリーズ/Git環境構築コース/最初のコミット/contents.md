# 最初のコミット

## このレッスンで学ぶこと

- Git がインストールされていることを確認する
- リポジトリを初期化する（`git init`）
- ファイルをステージングする（`git add`）
- 最初のコミットを作成する（`git commit`）

---

## 事前確認：Git は使える状態ですか？

コミット操作を始める前に、Git が正しくインストールされているか確認しましょう。
ターミナル（または SourceTree のターミナル機能）で以下を実行してください。

```bash
git --version
```

バージョン番号（例: `git version 2.43.0`）が表示されれば OK です。

> 表示されない場合は、IT ヘルプデスクへ「Git のインストール確認」として問い合わせてください。
> インストール手順は社内の「DX研修・日常開発向け 最低限ツールセットアップ手順」を参照してください。

---

## ステップ 1：作業フォルダを用意してリポジトリを初期化する

まず、練習用のフォルダを作成し、Git リポジトリとして初期化します。

```bash
mkdir my-first-repo
cd my-first-repo
git init
```

`Initialized empty Git repository in ...` と表示されれば成功です。

## ステップ 2：ファイルを作成する

リポジトリにファイルを追加してみましょう。
VS Code やテキストエディタで `README.md` を作成し、一行書いて保存します。

```bash
echo "# はじめてのリポジトリ" > README.md
```

ファイルが作成できたか確認します。

```bash
ls
```

`README.md` が表示されれば OK です。

---

## ステップ 3：ファイルをステージングする

Git では、コミットする前に「ステージング」という操作でコミット対象のファイルを選びます。

```bash
git add README.md
```

すべての変更ファイルをまとめてステージングしたい場合は次のコマンドも使えます。

```bash
git add .
```

ステージングの状態を確認するには `git status` を使います。

```bash
git status
```

`Changes to be committed:` のセクションに `README.md` が表示されていれば、ステージング完了です。

<!-- スクリーンショット: git status でステージング済みファイルが表示されている例 -->

> **SourceTree を使う場合**
> 「作業ツリーのファイル」エリアに表示されたファイルをチェックすることでステージングできます。
> GUI でコミット履歴やブランチ状態を確認したい方は SourceTree が便利です。

---

## ステップ 4：最初のコミットを作成する

ステージングしたファイルをコミットします。`-m` オプションでコミットメッセージを指定します。

```bash
git commit -m "最初のコミット"
```

以下のような出力が表示されれば成功です。

```
[main (root-commit) xxxxxxx] 最初のコミット
 1 file changed, 1 insertion(+)
 create mode 100644 README.md
```

<!-- スクリーンショット: git commit 実行後の出力例 -->

---

## ステップ 5：コミット履歴を確認する

コミットが記録されたか、ログで確認しましょう。

```bash
git log
```

コミットのハッシュ値・作成者・日時・メッセージが表示されます。

---

## まとめ

| 操作 | コマンド |
|---|---|
| リポジトリの初期化 | `git init` |
| ファイルのステージング | `git add <ファイル名>` |
| 状態の確認 | `git status` |
| コミットの作成 | `git commit -m "メッセージ"` |
| 履歴の確認 | `git log` |

最初のコミットができました。次のレッスンでは、ブランチの作成と切り替えを学びます。

## 確認ポイント

- [ ] `git --version` でバージョンが表示される

- [ ] `git commit` でコミットが作成できた
- [ ] `git log` でコミット履歴が確認できた
