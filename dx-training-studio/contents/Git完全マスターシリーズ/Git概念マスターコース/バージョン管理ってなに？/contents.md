## バージョン管理とは？

バージョン管理とは、ファイルの**変更履歴を記録・管理するしくみ**です。

たとえば、レポートを書いているとき、こんな経験はありませんか？

- `report_最終版.docx`
- `report_最終版2.docx`
- `report_本当に最終版.docx`

ファイル名で「バージョン」を管理しようとすると、すぐに混乱してしまいます。バージョン管理ツールを使えば、このような問題をスマートに解決できます。

## なぜバージョン管理が必要？

### 1. 過去のバージョンに戻せる

誤って大事なコードを削除してしまっても、いつでも以前の状態に戻せます。「元に戻す（Ctrl+Z）」の強力版だと思ってください。

### 2. 誰がいつ何を変更したか分かる

変更の記録（**コミット履歴**）を見れば、「この変更は誰が・いつ・なぜ行ったか」が一目でわかります。チームでの作業でも、問題の原因をすぐに特定できます。

### 3. 複数人で同じファイルを編集できる

チームメンバーがそれぞれ別の機能を同時に開発しても、変更を安全に統合できます。

## Git とは？

**Git** は、現在世界で最も広く使われているバージョン管理ツールです。

- 無料・オープンソース
- ローカル（自分のPC）で動作する
- チーム開発に欠かせないツール

### 社内での Git 環境

> **社内ワンポイント**
>
> 社内では Git のインストールに申請が必要です。インストーラには **SourceTree**（GUI ツール）が同梱されており、両方が一度にインストールされます。申請から完了まで数日かかる場合があるため、研修開始の **1週間前には申請**を出しておきましょう。
>
> インストール後、以下のコマンドでバージョンが表示されれば OK です。
>
> ```
> git --version
> ```
>
> 表示されない場合は IT ヘルプデスクへ問い合わせてください。
>
> 参考: [DX研修・日常開発向け 最低限ツールセットアップ手順](https://intranet.example.com/blog/dev-tools/setup-guide)
