    課題概要
        面倒な報告を自動化するツールを作る 
        提出期限

    解くべき課題
        会社で新しいプロジェクトが始まりツール環境が git/GitHub に変わりました
        ほとんどの人が git/GitHub には不慣れで理解度はバラバラ
        数十人の週会議で誰が・いつ・どのブランチをどのタイミングでマージするか、みたいな話をしています
        なかなか話がまとまらないし、進捗が分からないし、やるべきことが明確にならずに困っています
    作成したツールの概要
        やりたいことは、GitHub リポジトリの開発状況を分析し、レポートを作成し、関係者に自動で報告すること
        コミネコを参考にしながら、Python ベースで動作するツール `コミとら` を作りました
    4つのパーツ
        1. トリガー
            トリガー1
                push または merge をトリガーに CI ジョブを実行
                    + 手動実行も可能
                動作確認が目的で、python スクリプトのドライランのみ行います
                レポート作成、デプロイ、報告はやりません
            トリガー2
                毎朝7:00に本番ジョブを実行
                    + 手動実行も可能
                レポート作成、デプロイ、報告まで行います
        2. ソース元
            GitHub リポジトリ
            <https://github.com/ug-kitamura/AI_Driven_School>
        3. 処理する場所
            GitHub runner
        4. 届ける先
            レポートを GitHub Pages にアップロードします
            <https://ug-kitamura.github.io/AI_Driven_School/>
            さらに、レポートのサマリーと GitHub Pages のリンクを Slack に投稿します
    デザインの工夫
        会社のブランドガイドに則り、青・白・グレーを基調としたシンプルでモダンなデザインを目指しました
        コミとら君のロゴを Nano Banana を使って作りました
    ぶつかった課題と解決方法
        1. GitHub リポジトリの用意
            会社のリポジトリを使用するわけにもいかず...
            コミとら開発のために GitHub リポジトリを用意し、それ自身を分析対象としながら開発を進めました
        2. ブランチグラフの作成
            コミネコとの差別化を図るため、ブランチグラフを載せました
            mermaid を使用して実現しています
        3. Python 環境の構築
            はじめは Anaconda を使いましたが、仮想環境の立ち上げがやや面倒でした
            uv を採用することで効率化しました
        4. AI と Python の作業分担
            Deterministic（決定論的）な処理はなるべく Python 側で行い、
            Probabilistic（確率論的）な処理はなるべく AI スキル側で行うようにしました
            しかし AI と Python のデータ往復が増えるため、抽象化可能な箇所は AI スキルにまとめるなど工夫しました
