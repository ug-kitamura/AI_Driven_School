# training-editor-workspace-dark-mode Specification

## Purpose

DX Training Editor のダークテーマ時における Pane3 編集、曼陀羅、別シリーズコース選択、レッスンメタ・コースメタダイアログの視認性を定義する。`workspace-ui-polish` change により確定した挙動を正本とする。

## Requirements

### Requirement: ダークテーマ時の Pane3 編集は Cursor 風の配色とする

テーマが `dark` または `system` かつ OS がダークのとき、Pane3 編集モード（raw）の CodeMirror は **Cursor 現在のエディタ**に近い視認性で表示しなければならない（SHALL）。背景は深いグレー（VS Code dark 相当）とし、本文・見出し・コード・リンク・リスト・フロントマターが識別可能なコントラストでなければならない（SHALL）。行番号と折りたたみガターは控えめなグレーとし、ライト専用の固定色（例: `#237893`）をダークでそのまま用いてはならない（MUST NOT）。カーソル行のハイライトはダーク背景上で識別可能でなければならない（SHALL）。

#### Scenario: ダークで Markdown が読める

- **WHEN** テーマが `dark` である
- **AND** ユーザーが編集モードで見出し・リスト・コードを含むレッスンを開く
- **THEN** 背景は明るいグレー画面（`bg-background`）と区別できる暗色である
- **AND** シンタックス要素が 1 色のみのプレーンテキスト表示にならない

#### Scenario: ダークで行番号が控えめに表示される

- **WHEN** テーマが `dark` である
- **AND** ユーザーが編集モードを表示する
- **THEN** 行番号の色は本文より弱いコントラストのグレー系である

### Requirement: 曼陀羅の表示領域はダークで card 背景とする

グローバル曼陀羅（`GlobalHeader` ダイアログ）およびミニ曼陀羅（Pane2 サムネイル・拡大ダイアログ）の SVG コンテナは、`bg-white` 固定を用いてはならない（MUST NOT）。ダークテーマ時は `bg-card` 等のデザイントークンとし、Mermaid の描画設定はダーク向けの `themeVariables`（または `theme: "dark"`）を用いてノード・エッジ・文字が読めること（SHALL）。テーマ切替後に曼陀羅を開いたとき、最新の明暗設定で SVG が生成されなければならない（SHALL）。

#### Scenario: グローバル曼陀羅がダーク背景

- **WHEN** テーマが `dark` である
- **AND** ユーザーがグローバル曼陀羅を開く
- **THEN** グラフ周囲のコンテナ背景は白一色のままではない
- **AND** ノードラベルが背景に対して読める

#### Scenario: ミニ曼陀羅拡大がダーク背景

- **WHEN** テーマが `dark` である
- **AND** ユーザーが Pane2 のミニ曼陀羅を拡大表示する
- **THEN** 拡大ダイアログ内の背景はダーク UI と整合する

### Requirement: 別シリーズコース選択はダークで選択状態が識別できる

コースメタ編集内の別シリーズ前/次コース選択（`CrossSeriesCourseTreePicker`）は、ダークテーマ時にリスト領域が `bg-white` 固定であってはならない（MUST NOT）。選択中のコース行は、ダーク背景上で `bg-primary/5` 程度では識別できないため、十分なコントラスト（例: `bg-accent` またはより濃い primary 混色）でハイライトしなければならない（SHALL）。シリーズ見出し行もダークトークンで読めなければならない（SHALL）。

#### Scenario: ダークで別シリーズコースを選択できる

- **WHEN** テーマが `dark` である
- **AND** ユーザーがコースメタで別シリーズのコースを 1 件選択する
- **THEN** 選択行が未選択行と明確に区別できる

### Requirement: レッスンメタのタグ入力はダークトークンに合わせる

レッスンメタ編集のタグ入力コンポーネント（`LessonTagsInput`）の入力ボックスは `bg-white` 固定を用いてはならない（MUST NOT）。ダーク時は `bg-background` または Input と同系のトークンで、枠線・プレースホルダ・タグチップが読めなければならない（SHALL）。

#### Scenario: ダークでタグ欄の背景が統一される

- **WHEN** テーマが `dark` である
- **AND** ユーザーがレッスンメタのタグ欄を表示する
- **THEN** タグ入力ボックスの背景が周囲のダイアログと不自然な白抜けにならない

### Requirement: メタダイアログの入力コントロールはトークンベース背景とする

`META_DIALOG_CONTROL` およびそれを用いる Input / Select は、ライト・ダークの両方で `bg-white` 直書きに依存してはならない（MUST NOT）。`bg-background` 等の役割トークンを用い、ダーク時にコースメタ・レッスンメタ・シリーズ/コース追加ダイアログで入力欄が読めなければならない（SHALL）。

#### Scenario: ダークのコースメタで入力欄が読める

- **WHEN** テーマが `dark` である
- **AND** ユーザーがコースメタ編集ダイアログを開く
- **THEN** テキスト入力欄の背景が白固定ではない
