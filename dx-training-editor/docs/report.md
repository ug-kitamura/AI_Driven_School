    解くべき課題
        会社で新しいプロジェクトが始まりツール環境が git/GitHub に変わりました
        しかし、ほとんどの人が git/GitHub には不慣れで理解度はバラバラ
        多くの人にトレーニングをしなくてはならないが、教えられる人のリソースは限られている
        受講者が自分のペースで、レベルに合った内容を、独習形式で進められるコンテンツを作成したい
        補足
            私が AI Driven School の受講を決めた理由もこの課題を解決するためでした
    作成したツールの概要
        本気 AI ドリルの構成を参考に、トレーニングをシリーズ -> コース -> レッスンに分解しました。
        それらを各ペインで効率よく管理しながら作成する仕組みです。
        ペイン１
            シリーズと、その下の各コースの管理をします。
            シリーズとコースの追加・削除・並べ替えが可能です。
            シリーズ全体の進捗と、各コースの進捗を表示します。
        ペイン２
            コースと、その下の各レッスンの管理をします。
            コース概要の編集と、レッスンの追加・削除・並べ替えが可能です。
            コース全体の進捗と、各レッスンの進捗を表示します。
            ここでは各コースの受講順番を細かく設定可能です。
                複数のコースの修了が次のコースの受講条件になったり（合流）
                コースを修了したら複数のコースに進めるようになったり（分岐）
                シリーズ内だけでなく、シリーズをまたいだ複雑なフローを設定可能です。
                これを **DX トレーニング曼陀羅** と呼んでいます。
        ペイン３
            レッスンの原稿を編集します。
            原稿はマークダウン + YAML フロントマター形式を想定しています。
            YAML フロントマターに書かれたレッスンのメタ情報はモーダルでも設定可能にしています。
            編集モードはセクションごとに折りたたみ・展開が可能で、かつフォントサイズを Ctrl+マウスホイールで自由に変更できます。
            また、編集モードだけでなく、Preview 表示モード、差分表示モードなども用意しています。
            差分表示モードは git の HEAD との差分を表示します。
        ペイン４
            使用する画像を管理する画面です。
            1. 実際に使用している画像を管理するモード
            2. 画像を自分でアップロードするモード
            3. AI に画像を作らせるモード
            4. AI が web 検索してフリー画像サイトから画像を取得するモード
            を用意しています。
    工夫したポイント
        トレーニングコンテンツ作りで一番面倒なのが画像の作成と管理です。
        AI に画像を作らせたり、web 検索させたりすることが可能ですが、
        さらに一歩進めて、AI が実行するプロンプト自体を AI が自動で作る機能も用意しました。
        ペイン３の原稿内容とカーソル位置から、必要な画像を AI が自動で判断する仕組みです。
    苦戦したポイント
        1. ダークテーマの用意
            あったら面白いかも、と気楽に実装を始めたら結構はまりました。
            微妙にコントラストが低くて画面が見づらいなど、色味の細かな調整に苦労しました。
        2. DX トレーニング曼陀羅
            コースの順番を並べ替えた場合、それに応じて曼陀羅を書き換える必要があります。
            また、フローの入口側を更新したら、それに合わせて出口側の更新が必要です。（その逆も同様）
            さらに、矢印が２重で描画されるのを防いだり、循環フローになるケースを防いだりと、考えることが多すぎて
            これが一番苦心しました。
