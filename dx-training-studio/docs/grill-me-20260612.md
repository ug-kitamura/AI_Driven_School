## 決まったこと

### コンテンツ構造：フォルダ構成が正

レッスン本文は個別の `.md` ファイルとして保存し、フォルダ階層がシリーズ・コース・レッスンの構造を表す。

```
contents/
  _series-order.json          ← シリーズ表示順のみ（シリーズ名はフォルダ名）
  <シリーズ名>/
    01_<コース名>/
      _course.json            ← target_audience, prerequisites, next_courses
      01_<レッスン名>.md
      02_<レッスン名>.md
    02_<コース名>/
      ...
```

**設計方針の理由：**
- フロントマターが壊れていても、フォルダパスから構造が分かる
- 既存のmarkdown草稿を正しいフォルダに置くだけで読み込める
- mkdocsへの変換パイプラインとの相性が良い
- Gitでコンテンツ履歴を追える

**順序管理：**
- コースフォルダ・レッスンファイルに**数値プレフィックス**（`01_`, `02_`）で順序を表現
- UIで並び替えると、ファイル名・フォルダ名の番号が自動的に振り直される
- シリーズの順序はUIのみで管理し、`contents/_series-order.json` に保存

**リネームのルール：**
- Pane 1・Pane 2でシリーズ名・コース名・レッスン名を変更 → フォルダ名・ファイル名を変更
- フロントマターの値を直接書き換えても → フォルダ名・ファイル名は変更しない（一方通行）

**曼陀羅データ：**
- 独立したストレージは不要。フォルダ構成 + `_course.json` の `prerequisites` / `next_courses` から導出

**アプリ名・アイコン：** コードに固定（設定ファイル不要）

### 複数トレーナーの協業モデル

- GitHub リポジトリで非同期協業（Git push/pull）
- 各トレーナーがローカルでアプリを起動して編集
- リアルタイム同時編集（Google Docs 的）は対象外

### 保存先の決定（ローカル版・クラウド版）

#### コンテンツ・構造データ

| 対象 | ローカル用 | クラウド用 |
|---|---|---|
| レッスン本文（markdown） | ファイルシステム `.md`（自動保存） | Neon DB（ドラフト）→ GitHub API（コミット時） |
| コース構造（`_course.json`） | ファイルシステム（自動保存） | Neon DB（ドラフト）→ GitHub API（コミット時） |
| シリーズ表示順（`_series-order.json`） | ファイルシステム（自動保存） | GitHub API（コミット時のみ） |

#### メディア

| 対象 | ローカル用 | クラウド用 |
|---|---|---|
| 正規画像（プロモート済み） | **Vercel Blob**（ローカルから直接） | Vercel Blob（同じ） |
| ステージング画像（未プロモート） | ローカル一時フォルダ | ローカル一時フォルダ or Blob |

#### アプリ設定

| 対象 | ローカル用 | クラウド用 |
|---|---|---|
| エディタ設定（テーマ・フォントなど） | localStorage | localStorage |
| アプリ名・アイコン | コードに固定 | コードに固定 |

#### データベース（Vercel Neon）

| 対象 | ローカル用 | クラウド用 |
|---|---|---|
| ステータス変更履歴（`lesson_status_log`） | **Neon** | **Neon**（同じ） |
| コンテンツドラフト（`drafts`） | 不要（fsに保存） | **Neon** |

---

### 保存タイミング

| 操作 | ローカル | クラウド |
|---|---|---|
| **自動保存** | `fs.writeFile`（800ms debounce） | Neon DB（800ms debounce） |
| **コミット** | `git commit`（shell実行） | Neon ドラフト → GitHub API |

**「コミット」ボタンはローカル・クラウド共通のUI**として実装する。内部実装のみ環境で切り替わる（アダプターパターン）。

---

### ステータス変更履歴の表示場所

グローバルヘッダーに「アクティビティ履歴」ボタンを追加し、ダイアログで全レッスン横断の履歴を表示する。

```
[DXトレーニング曼陀羅]  [アクティビティ履歴]  [設定]
```

## 今回の実装課題（3つ）

### 課題1：Vercel Blob — 画像保存（ローカル・クラウド共通）

既存の画像APIルート（`/api/images/upload`、`/api/images/promote` など）をVercel Blobに切り替える。ローカルから直接Blobへアップロード・参照できるようにする。

### 課題2：Vercel Neon — ステータス変更履歴（ローカル・クラウド共通）

`lesson_status_log` テーブルを作成。UIでレッスンのステータスを変更したタイミングでNeonに記録し、グローバルヘッダーのアクティビティ履歴ダイアログで表示する。

### 課題3：Vercel Neon + GitHub API — ドラフト保存（クラウド版のみ）

`drafts` テーブルを作成。クラウドエディタとして動作するとき、編集内容をNeonにドラフト保存し、「コミット」ボタンでGitHub APIを通じてリポジトリに反映する。

## テーブル設計

### `lesson_status_log`

```
┌─────────────────────────────────────────────────────────┐
│                    lesson_status_log                     │
├──────────────┬──────────────┬──────────────────────────┤
│ カラム       │ 型           │ 説明                      │
├──────────────┼──────────────┼──────────────────────────┤
│ id           │ SERIAL PK    │ 自動採番                  │
│ lesson_id    │ TEXT NOT NULL│ レッスンID（フロントマター）│
│ old_status   │ TEXT NOT NULL│ "open"|"in_progress"|"done"│
│ new_status   │ TEXT NOT NULL│ "open"|"in_progress"|"done"│
│ author       │ TEXT         │ フロントマターの author    │
│              │              │ （複数の場合もそのまま）   │
│ changed_at   │ TIMESTAMPTZ  │ DEFAULT now()             │
└──────────────┴──────────────┴──────────────────────────┘
```

```sql
CREATE TABLE lesson_status_log (
  id          SERIAL PRIMARY KEY,
  lesson_id   TEXT NOT NULL,
  old_status  TEXT NOT NULL,
  new_status  TEXT NOT NULL,
  author      TEXT,
  changed_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### `drafts`

```
┌─────────────────────────────────────────────────────────┐
│                        drafts                            │
├──────────────┬──────────────┬──────────────────────────┤
│ カラム       │ 型           │ 説明                      │
├──────────────┼──────────────┼──────────────────────────┤
│ id           │ SERIAL PK    │ 自動採番                  │
│ draft_type   │ TEXT NOT NULL│ "lesson"|"course"         │
│              │              │ |"series_order"           │
│ entity_id    │ TEXT NOT NULL│ lesson_id / course_id     │
│              │              │ / series_id               │
│ content      │ TEXT NOT NULL│ .md本文 or JSON文字列     │
│ updated_by   │ TEXT         │ フロントマターの author    │
│ updated_at   │ TIMESTAMPTZ  │ DEFAULT now()             │
├──────────────┴──────────────┴──────────────────────────┤
│ UNIQUE (draft_type, entity_id)                          │
└─────────────────────────────────────────────────────────┘
```

```sql
CREATE TABLE drafts (
  id          SERIAL PRIMARY KEY,
  draft_type  TEXT NOT NULL CHECK (draft_type IN ('lesson', 'course', 'series_order')),
  entity_id   TEXT NOT NULL,
  content     TEXT NOT NULL,
  updated_by  TEXT,
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (draft_type, entity_id)
);
```

**設計方針：**
- `UNIQUE (draft_type, entity_id)` → 同じエンティティのドラフトは常に1行（UPSERTで上書き）
- `content` はTEXT統一：lesson は生のmarkdown、course/series_order はJSON文字列
- `author` はFK制約なし（将来ユーザーテーブル追加時にマイグレーションで対応）

### テーブル間の関係

```
  lesson_status_log              drafts
  ┌─────────────────┐           ┌─────────────────┐
  │ id (PK)         │           │ id (PK)         │
  │ lesson_id ──────┼───────────┼── entity_id     │
  │ old_status      │  ※ FK無し │ draft_type      │
  │ new_status      │  (TEXT参照)│ content         │
  │ author          │           │ updated_by      │
  │ changed_at      │           │ updated_at      │
  └─────────────────┘           └─────────────────┘

  ※ lesson_id / entity_id はファイルシステムの
    フロントマター id と対応（DB外部で管理）
```

## 詰め切れていない事項

| 論点 | 内容 |
|---|---|
| **移行スクリプト** | 既存の `data/content.json` から新しいフォルダ構成（`contents/<シリーズ>/<コース>/<レッスン>.md`）への変換ツールが必要 |
| **Blob内の画像URL参照** | markdown本文内の画像参照を相対パス（`images/file.png`）のままにするか、Blob URL（`https://...vercel-storage.com/...`）に変えるかを決める必要がある |
| **GitHub OAuth** | クラウド版でGitHub APIを呼ぶ際の認証方法（NextAuth.js 等）は未設計 |
| **ゲーミフィケーション用テーブル** | `users`（受講者）・`progress`・`trophies`・`rankings`・`streaks` は将来フェーズで設計する |
