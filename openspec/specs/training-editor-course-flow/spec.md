## Requirements
### Requirement: シリーズ内のコース順序は配列順である

シリーズ内のコース間の受講順序は、`series.courses` の並びのみで定義されなければならない（SHALL）。Pane1 のドラッグ＆ドロップによる並べ替えは、その配列を更新し、シリーズ内の受講順も更新しなければならない（SHALL）。

#### Scenario: 並べ替えで受講順が更新される

- **WHEN** ユーザーが Pane1 でシリーズ内のコースを並べ替える
- **THEN** `series.courses` の配列順が新しい一覧順と一致する
- **AND** そのシリーズのシリーズ内受講順が新しい順序と一致する

#### Scenario: 先頭コースにシリーズ内の前コースはない

- **WHEN** コースが `series.courses` の先頭である
- **THEN** 配列順から導出されるシリーズ内の前コースは存在しない

### Requirement: 別シリーズのリンクはコースメタにのみ保存する

各コースの `prerequisites` および `next_courses` には、当該コースの親シリーズ以外に属するコース ID のみを保存しなければならない（SHALL）。エディタは、これらの配列にシリーズ内の隣接コース ID を永続化してはならない（MUST NOT）。

#### Scenario: 保存時に同シリーズ ID を除外する

- **WHEN** ユーザーが前提・次コースを含むコースメタを保存する
- **THEN** 同一親シリーズ内のコースを指す ID は保存配列から除外される

#### Scenario: 読み込み時にレガシーな同シリーズ ID を正規化する

- **WHEN** ワークスペースデータを読み込み、コースメタ配列に当該コースと同じシリーズの ID が含まれる
- **THEN** UI で使う前にそれらの ID が除去される

#### Scenario: 並べ替えは別シリーズメタを変えない

- **WHEN** ユーザーがシリーズ内のコースのみを並べ替える
- **THEN** 全コースの `prerequisites` と `next_courses` は、読み込み時正規化を除き変更されない

### Requirement: コースメタ編集モーダルの項目

コースメタ編集ダイアログは、次の項目を提供しなければならない（SHALL）: 編集可能な受講対象者（単一行テキスト）; 読取専用の同シリーズの前コース（最大1件、`i-1` の日本語表示名）; 読取専用の同シリーズの次コース（最大1件、`i+1` の日本語表示名）; 編集可能な別シリーズの前コース（0件以上、日本語名で多選択、`prerequisites` に ID 保存）; 編集可能な別シリーズの次コース（0件以上、多選択、`next_courses` に ID 保存）。

#### Scenario: 同シリーズの前コースを読取表示する

- **WHEN** シリーズ内で先頭以外のコースについてメタダイアログを開く
- **THEN** ダイアログに `series.courses[i-1]` の日本語名が同シリーズの前コースとして表示される
- **AND** その項目は編集できない

#### Scenario: シリーズ先頭では同シリーズの前コースなし

- **WHEN** シリーズ先頭のコースについてメタダイアログを開く
- **THEN** 同シリーズの前コース欄はなし（例: 「なし」）と表示される

#### Scenario: 別シリーズの選択は ID として保存する

- **WHEN** ユーザーがメタダイアログで別シリーズのコースを選択して保存する
- **THEN** `prerequisites` と `next_courses` には選択した別シリーズのコース ID のみが含まれる

### Requirement: ミニ曼陀羅のローカルグラフ

コースが選択されているとき、Pane2 は現在コースを中心としたミニ曼陀羅（Mermaid）を描画しなければならない（SHALL）。表示内容は次のとおり: シリーズ内の前ノードは最大1件; シリーズ内の次ノードは最大1件; `prerequisites` に列挙された別シリーズのコースはすべて; `next_courses` に列挙された別シリーズのコースはすべて。シリーズ内のエッジは、該当ノードがあるとき `前 → 現在 → 次` としなければならない（SHALL）。別シリーズのエッジは、リンクごとに `前提 → 現在` および `現在 → 次` としなければならない（SHALL）。

#### Scenario: ミニグラフにシリーズ内の前後を含む

- **WHEN** 選択コースがシリーズ内で index `i` にあり、`i > 0` かつ `i < length - 1` である
- **THEN** ミニ曼陀羅に `courses[i-1]` と `courses[i+1]` のノードが現在コースへつながって表示される

#### Scenario: ミニグラフに別シリーズリンクをすべて含む

- **WHEN** 選択コースのメタに別シリーズの前提コースが2件、次コースが1件ある
- **THEN** ミニ曼陀羅に前提2ノードから現在へのエッジと、現在から次1ノードへのエッジが表示される

#### Scenario: ミニグラフのノードクリックでコース選択

- **WHEN** ユーザーがミニ曼陀羅（サムネイルまたは拡大モーダル）で現在以外のノードをクリックする
- **THEN** ワークスペースがそのコース ID を選択する

### Requirement: グローバル曼陀羅はコース順序を反映する

グローバル曼陀羅（`GlobalHeader` ダイアログ）は、各シリーズの隣接ペアごとに `courses[i-1]` から `courses[i]` へのエッジを描画しなければならない（SHALL）。別シリーズのコースに解決する `prerequisites` の各エントリについて `前提 → 当該` のエッジを、`next_courses` の各エントリについて `当該 → 次` のエッジを描画しなければならない（SHALL）。同一のコース対 `(from, to)` について、両配列から同じ向きのリンクが得られる場合はエッジを1本にまとめなければならない（SHALL）。`series` が変更されたとき（シリーズ内の並べ替えを含む）、次回描画で使う曼陀羅定義は、シリーズ内の鎖について更新後の配列順を反映しなければならない（MUST）。

#### Scenario: グローバル曼陀羅にシリーズ内の鎖がある

- **WHEN** シリーズのコースが配列順で `[A, B, C]` である
- **THEN** グローバル曼陀羅の当該サブグラフ内に `A → B` と `B → C` のエッジがある

#### Scenario: 並べ替え後にグローバル曼陀羅が更新される

- **WHEN** ユーザーがシリーズ内でコースを並べ替え、`series` 状態が更新されたうえでグローバル曼陀羅を開くまたは再描画する
- **THEN** シリーズ内エッジが新しい `courses[]` の順序と一致する

#### Scenario: 別シリーズの前提コースがグローバル曼陀羅に表示される

- **WHEN** コース X の `prerequisites` に別シリーズのコース P がある
- **THEN** グローバル曼陀羅に `P → X` のエッジがある

#### Scenario: 双方向メタでも別シリーズエッジは1本

- **WHEN** コース A の `next_courses` に B があり、かつコース B の `prerequisites` に A がある（いずれも別シリーズ）
- **THEN** グローバル曼陀羅に `A → B` のエッジは1本だけある

#### Scenario: 並べ替えでも別シリーズエッジは維持される

- **WHEN** コース X の `next_courses` に別シリーズのコース Y があり、ユーザーは X の属するシリーズ内のみ並べ替える
- **THEN** グローバル曼陀羅に `X → Y` のエッジが残る

### Requirement: 別シリーズリンク保存時の双方向同期

コースメタ保存時、エディタは編集対象コースの別シリーズ `prerequisites` および `next_courses` を正規化したうえで、ワークスペース内の関連コースを更新し、次を満たさなければならない（SHALL）:

- **ミラー**: 別シリーズのコース `X` から `Y` へのリンクが存在するとき、`X.next_courses` に `Y` が含まれ、かつ `Y.prerequisites` に `X` が含まれる。
- **ソースシリーズ排他**: 任意のターゲットコース `Y` とソースシリーズ `S` について、`Y.prerequisites` に含まれる `S` 内のコースは高々1つである。
- **ターゲットシリーズ排他（出口）**: 任意の出口元コース `X` とターゲットシリーズ `T` について、`X.next_courses` に含まれる `T` 内のコースは高々1つである。`X → Y`（`Y` は `T` 内）を新たに確立するとき、同じ `T` 内の他コース `Z` への `X.next_courses` および `Z.prerequisites` 内の `X` は除去されなければならない（SHALL）。これにより、コースメタ picker の「各シリーズから1コースまで」と保存後の `next_courses` が一致する。
- **編集優先**: ユーザーがコース `E` のメタを保存し、`E` がシリーズ `S` からターゲット `Y` への別シリーズリンクを設定した場合、`S` 内で `Y` への唯一のゲートウェイは `E` となる。同シリーズ `S` の他コースに残っていた `Y` への `next_courses` および `Y.prerequisites` 内の `S` 由来の古い ID は除去されなければならない（SHALL）。
- **多シリーズ合流**: 上記排他はソースシリーズごとに適用される。異なるソースシリーズから同一ターゲット `Y` へのリンク（例: `A1 → Y` と `B1 → Y`）は同時に存在してよい（SHALL）。

リンク削除時、エディタは除去された向きのミラー側エントリも削除しなければならない（SHALL）。削除に伴い他コースへリンクを自動付け替えてはならない（MUST NOT）。

#### Scenario: 同シリーズ内の出口を付け替える

- **WHEN** シリーズ A に `A1`, `A2`、シリーズ B に `B1` があり、当初 `A2.next_courses` に `B1`、`B1.prerequisites` に `A2` がある
- **AND** ユーザーが `A1` のメタを保存し、別シリーズの次コースに `B1` を設定する
- **THEN** `A1.next_courses` に `B1` が含まれる
- **AND** `A2.next_courses` から `B1` が除去される
- **AND** `B1.prerequisites` はシリーズ A 由来として `A2` の代わりに `A1` のみを含む

#### Scenario: 異なるソースシリーズからの合流は維持する

- **WHEN** `A1.next_courses` に `B2`、`B1.next_courses` に `B2` があり、それぞれミラーが整合している
- **AND** ユーザーが `A2` のメタを保存し、別シリーズの次コースに `B2` を設定する
- **THEN** `A2.next_courses` に `B2` が含まれ、`A1.next_courses` から `B2` が除去される
- **AND** `B2.prerequisites` のシリーズ A 由来は `A2` のみとなる
- **AND** `B1.next_courses` および `B2.prerequisites` のシリーズ B 由来 `B1` は変更されない

#### Scenario: prerequisites 側の編集でもミラーと排他が成立する

- **WHEN** `A2.next_courses` に `B1`、`B1.prerequisites` に `A2` がある
- **AND** ユーザーが `B1` のメタを保存し、別シリーズの前コースに `A1` を設定する
- **THEN** `A1.next_courses` に `B1` が含まれ、`A2.next_courses` から `B1` が除去される
- **AND** `B1.prerequisites` のシリーズ A 由来は `A1` のみとなる

#### Scenario: 入口編集で同一ターゲットシリーズへの古い出口が除去される

- **WHEN** シリーズ A に `A1`, `A2`、シリーズ B に `B1`, `B2` があり、当初 `A1.next_courses` に `B1`、`A2.next_courses` に `B2` でミラーが整合している
- **AND** ユーザーが `B2` のメタを保存し、別シリーズの前コースを `A2` から `A1` に変更する
- **THEN** `A1.next_courses` に `B2` が含まれ、同シリーズ B 向けの `B1` は含まれない
- **AND** `A2.next_courses` から `B2` が除去される
- **AND** `B1.prerequisites` から `A1` が除去される
- **AND** `B2.prerequisites` のシリーズ A 由来は `A1` のみとなる

#### Scenario: リンク削除でミラーが除去される

- **WHEN** `A1.next_courses` に `B1`、`B1.prerequisites` に `A1` がある
- **AND** ユーザーが `A1` のメタを保存し、別シリーズの次コースから `B1` を外す
- **THEN** `A1.next_courses` に `B1` が含まれない
- **AND** `B1.prerequisites` から `A1` が除去される
- **AND** 他コースに `B1` への新しいリンクが自動追加されない

#### Scenario: UI の同一シリーズ1選択制約と保存結果が一致する

- **WHEN** コースメタ編集ダイアログで別シリーズの前/次コースを選択する
- **THEN** 各 picker 内では同一シリーズから高々1コースのみ選択可能である（既存 UI 動作）
- **AND** 保存後、任意のコース `X` の `next_courses` において、同一ターゲットシリーズからの ID は高々1つである
- **AND** 保存後のワークスペース状態は上記ミラー・排他・編集優先の Requirement を満たす

### Requirement: コースメタ保存前の曼陀羅循環チェック

コースメタ保存前、エディタは編集内容を `applyCrossSeriesCourseMetaEdit` と同等のドライランで反映したうえで、曼陀羅全体の有向グラフ（シリーズ内の `courses[]` 鎖および別シリーズの `prerequisites` / `next_courses` リンク）に循環が生じるかを評価しなければならない（SHALL）。グラフ構築および巡回判定は、グローバル曼陀羅描画と同型の辺集合を用いなければならない（SHALL）。

別シリーズリンクの変更により、ドライラン後のグラフに新たな循環が生じる場合、コースメタ編集ダイアログは保存前に警告を表示し、保存を完了してはならない（MUST NOT）。警告は別シリーズの前/次コース設定の見直しを促す内容としなければならない（SHALL）。別シリーズリンクを変更せず、当該コースの名称や受講対象者のみを保存する場合、既存データに循環があっても保存をブロックしてはならない（MUST NOT）。

#### Scenario: 循環を生む別シリーズリンク変更は保存できない

- **WHEN** ユーザーがコースメタ編集ダイアログで別シリーズの前/次コースを変更し、保存を試みる
- **AND** ドライラン後の曼陀羅全体に循環する経路が存在する
- **AND** 変更前のワークスペース状態には循環がなかった、または当該コースの別シリーズリンクが変更されている
- **THEN** ダイアログに循環に関する警告が表示される
- **AND** 保存は実行されず、ダイアログは開いたままである

#### Scenario: 非リンク項目のみの保存は循環があってもブロックしない

- **WHEN** ワークスペースの曼陀羅全体に既に循環がある
- **AND** ユーザーが当該コースの別シリーズ前/次を変更せず、コース名または受講対象者のみを保存する
- **THEN** 循環警告は表示されない
- **AND** 保存が完了する

#### Scenario: 循環のない変更は通常どおり保存できる

- **WHEN** ユーザーが別シリーズリンクを変更して保存を試みる
- **AND** ドライラン後の曼陀羅全体に循環がない
- **THEN** 警告は表示されない
- **AND** リンク同期 Requirement に従って保存が完了する

### Requirement: 別シリーズコース選択 UI はダークで十分なコントラストを満たす

コースメタ編集ダイアログ内の別シリーズ前/次コース選択 UI（`CrossSeriesCourseTreePicker`）は、ダークテーマ時にリスト背景・シリーズ見出し・選択行・ホバー行が識別可能なコントラストでなければならない（SHALL）。詳細は `training-editor-workspace-dark-mode` の曼陀羅・ピッカー要件と整合させなければならない（SHALL）。

#### Scenario: ダークでチェック済みコースが見える

- **WHEN** テーマが `dark` である
- **AND** ユーザーが別シリーズのコースを 1 件以上選択している
- **THEN** チェック済み行が未選択行と明確に区別できる
